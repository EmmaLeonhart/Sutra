\begin{abstract}
Frozen general-purpose language-model embedding spaces encode
relational structure as vector arithmetic — a property established
across the knowledge-graph-embedding literature (TransE, RotatE,
the word-analogy line). Taking that as given, this paper presents
the design and implementation of \ifanon{}a programming language\else{}\textbf{Sutra}\fi,
a typed, purely functional programming language whose compile target
is a single tensor-op graph over a frozen LLM embedding substrate.
The contribution is algorithmic: a consolidated set of vector-symbolic
primitives (bind, unbind, bundle, similarity, rotation, soft-halt RNN
cells) that work on natural anisotropic embedding spaces where the
textbook Hadamard-product VSA fails, plus a compiler that lowers the
whole program to one fused tensor-op graph.
\ifanon{}The system\else{}Sutra\fi{} is a working compiler today:
parser, type checker, codegen, runtime; three demonstration programs
plus loop demonstrations execute end-to-end with all expected outputs
correct. The language also positions learned-matrix binding as an
explicit extension point, with rotation binding shipped today and
learned matrices as the natural next implementation. We give an
honest account of which substrate-purity claims are shipped and
which remain pending fixes.
\end{abstract}

\section*{Why this is a skeleton, not a finished LaTeX paper}

For \texttt{paper.md} on clawRxiv, the Markdown source is what
ships and is complete. For NeurIPS, a year-specific LaTeX template
is required and the year's deadline+template aren't confirmed yet.
Rather than guess the template (NeurIPS evolves it annually),
this skeleton stages the anon-macro infrastructure so the future
template-integration step is a short edit, not a full rewrite.
