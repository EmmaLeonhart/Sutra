## The core point (Emma 2026-06-17): memoization IS how Sutra gets native recursion

**Sutra has no native runtime recursion** — a function cannot call itself on the substrate (the V1
codegen only does direct VSA-builtin calls + tail-recursive loops). That is *exactly why* tier 4
exists: **memoization, in the explicit-work-stack-as-a-loop form, is the mechanism that GIVES Sutra
native recursion.** It is not a partial optimization for some cases — it is the general way every
multiple recursion runs natively. "Memoize everything" is the instruction, and it is realizable.

## The general mechanism: explicit-work-stack + memo, run as a loop

Any multiple recursion `f(x) = combine(f(a(x)), f(b(x)), …)` is transformed into a **tail-recursive
loop** that carries two pieces of state: an explicit **agenda** (the stack of pending subproblems —
what the call stack held implicitly) and a **memo table** (already-computed results). The implicit
call stack moves into ordinary substrate memory (a value), so no native call stack is needed:

```
solve(x):                          # driver
  agenda = [x]; memo = {}
  loop while agenda not empty:     # a while_loop  ->  recurrent neurons (tier 1, NATIVE)
    t = top(agenda)
    if t in memo: pop(agenda); continue
    subs = children(t)             # the recursive arguments a(t), b(t), …
    ready = all (s in memo) for s in subs
    if ready: memo[t] = combine(memo[s] for s in subs); pop(agenda)
    else:     push(agenda, the subs not yet in memo)
  return memo[x]
```

- The loop is a `while_loop` → recurrent neurons → **native, no call stack, no WASM**.
- The **agenda** and **memo** are values carried as recurrent (`slot`) loop state — "the memo table
  is the state, growing over time," fitting Sutra's stateful-program-as-time-series model exactly.
- **Memoize everything:** the memo is always present. Overlapping subproblems (fib) are computed
  once → the call *tree* flattens to a **DAG** → exponential becomes linear. Non-overlapping
  subproblems (a genuinely tree-shaped traversal) get no reuse speedup but **still run natively** via
  the same loop — memoization costs nothing extra and never forces a WASM jump.

## Tabulation is an OPTIMIZATION of this (not a separate path)

For recursion over a regular integer index (`fib(n)` over `0..n`), the agenda is trivial — the
subproblems are just `0,1,2,…,n` in order — so the explicit agenda collapses into a simple bottom-up
counter loop filling the memo (`dict<int,int>` / array): classic tabulation. That is a *special-case
optimization* of the general work-stack loop (skip the agenda when the subproblem order is
statically known), not a different mechanism and not the boundary of what's native.

## Building blocks (confirmed 2026-06-17)

- **`while_loop` → recurrent neurons** with carried `slot` state (the `sutra-from-ocaml` `while_sum`
  fixture). This is the loop.
- **`dict<int,int>`** (substrate-routed scalar dict) + the rotation-hashmap (`map<K,V>`) — the memo
  table (and the agenda, for irregular keys).
- Arrays / the RAM device — the agenda stack for the general form.
- **No native recursion** — which is the reason this transform exists, not a limitation that pushes
  work to WASM.

## Bounded sub-steps (each compile-AND-run, substrate-verified)

1. **4a — demonstrate the native target.** Hand-write the memoizing loop for a recursion (start with
   the tabulation special case: a `while_loop` filling a memo for `fib`, then the general
   work-stack form for a small irregular recursion) in `.su`, run on the substrate, confirm
   `== ground truth`. Proves multiple recursion runs natively via memoization-as-a-loop — no
   recursion, no WASM. Grounds 4b.
2. **4b — the automatic transform.** A compiler pass that rewrites a multiple-recursive function into
   the work-stack + memo loop (with the tabulation collapse for index-structured cases). Detect the
   recursive shape (base case + recursive calls + combine), synthesize the agenda/memo loop, emit a
   `while_loop`. Verify across shapes (`fib`, a tree-recursive sum, a multi-arg recursion) — each
   runs natively == reference. This is the substantive analysis.

## Tier 5 (WASM) after tier 4

Once tier 4 makes recursion native via memoization, **WASM is no longer the recursion fallback at
all.** WASM (tier 5, the completed `wasm_core`) is reserved for genuinely imperative / `eval` / FFI
constructs — a much smaller, principled category. (The `wasm_core` running recursive `fib` was the
*interim* proof that recursion runs at all; tier 4 makes it native.)

## Why scope before coding

4b (automatically rewriting arbitrary multiple recursion into the agenda+memo loop, with the
tabulation collapse) is a real, substantive compiler transform with no existing scaffolding.
Decomposing 4a (prove the native loop target by hand) → 4b (the automatic transform) keeps each step
bounded and substrate-verified, and — corrected here — keeps the scope at "memoize everything →
native," not a tabulation-only carve-out.
