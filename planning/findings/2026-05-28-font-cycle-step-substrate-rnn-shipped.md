# 2026-05-28 — font.su cycle_step is now a real substrate-state RNN (Option B)

Closes the blocker in `planning/findings/2026-05-28-cycle-step-rewrite-blocked.md`.
Emma's `AskUserQuestion` choice was Option B (matrix-lookup softmax / pure
substrate), then "add matrix-literal support" as its prerequisite. Both shipped.

## What shipped

`cycle_step(vector typed_onehot, scalar has_typed) -> vector` in `font.su`
(generated by `demos/font/generate_font_su.py` → `gen_cycle_step`).

- **State = a 36-dim one-hot** over the glyph cycle `ABC…Z0…9`, held in a
  `recurring vector glyph` slot that lives ON THE SUBSTRATE across ticks
  (`recur`, non-halting-loop.md). It is NOT a scalar char_code extracted to the
  host and fed back — the prior host-state-shuttle shape that failed the
  substrate-RNN claim.
- **Advance = one matmul** `advanced = Tensor.MatrixMul(P, glyph)` where `P` is
  the frozen 36×36 cyclic-shift permutation matrix, built with the
  `matrix_literal` source-level primitive (shipped same day). `(P @ e_i) =
  e_{(i+1) mod 36}`, so the one-hot advances by one and wraps Z→0, 9→A.
- **Typed override** = substrate weighted sum
  `next = advanced*(1 - has_typed) + typed_onehot*has_typed`. The host builds
  the typed one-hot from a keypress (an INPUT crossing the boundary, not state).
- **Render** decodes the returned one-hot to a char only for display — a
  monitoring boundary, like `count.su` decoding its count vector. The
  recurrence never leaves the substrate.

## Verification (measured, on the substrate)

`demos/font/test_font_cycle.py` (rewritten; 4 tests, compiles cycle_step in
isolation — see "Compile cost" below):

- **Advance**: 38 ticks from initial 'A' → `BCDEFGHIJKLMNOPQRSTUVWXYZ0123456789ABC`
  — every single-step advance, both wraps (Z→0, 9→A), and the full-loop return.
- **Typed override**: `cycle_step(onehot('Q'), 1.0)` → Q; the next advance → R
  (state became Q on the substrate).
- **has_typed=0 ignores the typed one-hot**: from 'A' → 'B' regardless.
- **Signal-separation gap = 1.0** (CLAUDE.md "Subtler substrate breaches" #3):
  the recurring state is bit-exact one-hot every tick — active weight = 1.0,
  every other entry = 0.0, sum = 1.0, no drift across 40 ticks. `gap =
  min(active) − max(non-active) = 1.0 − 0.0 = 1.0`.

State-locus audit: the recurring slot is a vector surviving across calls with
no host `real()` between ticks — a genuine substrate RNN, not a counter.
Dim audit: state/advance are 36-d problem-sized (zero `basis_vector` calls), so
cycle_step runs at `runtime_dim=8`; the old "766 of 768 dims dead weight"
critique no longer applies.

## Sub-fix: matrix_literal egglog-simplify cost (65s → 0.84s)

Building the 36×36 `P` as `matrix_literal(vector_literal(...)×36)` is ~1300
literal nodes. The `_egglog_post_pass` in `simplify.py` ran egglog
equality-saturation on every subexpression, lifting the entire literal tree
into egglog (redundantly at each nesting level) — **65s for the cycle_step
function alone**. Fix: skip the egglog post-pass for `vector_literal` /
`matrix_literal` Call nodes — pure float-literal args, nothing to algebraically
simplify. `simplify_module` on the cycle_step dropped **65.09s → 0.84s**. All
89 simplify/egglog/literal tests still pass (the guard only removes work, never
changes a non-literal result).

## Flagged (PRE-EXISTING, not introduced here): font.su glyph compile is >300s

The full `font.su` compile is dominated by a PRE-EXISTING cost unrelated to
cycle_step: the 36 `bit_<C>` 25-way `select`s + the 36-way `glyph_pixel`
`select`, each lifted into the egglog post-pass. With cycle_step stubbed out,
`simplify_module` on the glyph functions alone still **exceeds 300s**. This is
masked in normal use by an on-disk compile cache (the `.font.compiled-*.py`
files); editing font.su invalidates the cache and pays the >300s once.

This is why `test_font_cycle.py` now compiles cycle_step in ISOLATION (12s for
4 tests) instead of the full font.su via `font_demo._compile_cycle`. The same
egglog-literal-constructor fix above would likely help the `select([...],
[make_real(c0), …])` glyph case too (the branches are `make_real` literal
constructors), but generalizing the skip to `make_real`/`select` is riskier
(select can be legitimately simplified) and is left as a separate, measured
follow-on — see queue.md.
