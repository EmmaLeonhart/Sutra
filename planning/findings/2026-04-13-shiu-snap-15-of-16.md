# Snap on real W: 15/16 with random-neuron inputs

**Date:** 2026-04-13
**Script:** `fly-brain/shiu_snap_test.py`
**Substrate:** Shiu LIF 138,639 neurons / 15,091,983 synapses, real
FlyWire v783 W, no polar decomposition.
**Question:** Can a codebook-cleanup snap (argmax cosine over a K-class
codebook of prototype responses) correctly classify a fresh-seed repeat
of one of the prototype inputs?
**Answer:** 15/16 correct (~94%) with 16 disjoint random 50-neuron
input patterns. Single failure is a random-input overlap artifact, not
a dynamics or noise failure.

## Setup

- 16 disjoint 50-neuron input patterns p_0..p_15 drawn uniformly at
  random from 138,639 neurons (total 800 driven neurons across classes;
  no overlap between classes).
- Codebook compile: run each pattern once at seed 2000+k, record
  per-neuron spike-count vector as prototype out_k (dimension 138,639).
- Query: for each class k, run the same pattern p_k at seed 5000+k
  (different Poisson stream), compute cos(query, prototype_j) for all
  j, predict argmax_j. Correct iff argmax == k.
- Drive 200 Hz, 100 ms, PyTorch CUDA on 4070 Laptop.
- Compile 16.6 s, query 16.4 s, total ~35 s wall.

## Interpretation

The gap distribution is bimodal. 12 of 16 classes have best-vs-second-best
gaps above 0.95 — near-orthogonal responses, decisive argmax. 4 classes
(4, 5, 6, 11) have small gaps (0.03–0.10), and one of those (class 4)
flips to predict class 11 instead of class 4. The gap at class 4 is
0.0354, while the Poisson-stability noise floor is ~0.03–0.04 (from the
1 − 0.96 stability ceiling measured in the substrate probe). This means
the misclassification is not primarily a Poisson-noise event — it is
the signature of *two random 50-neuron input patterns whose downstream
response vectors happen to be nearly collinear*.

Where does the collinearity come from? With 50 neurons sampled from
138,639, the probability of any two random subsets sharing significant
downstream structure is small but nonzero — the brain has dense hubs
(KCs, MBONs, CX ring neurons) that many upstream neurons project to.
If patterns p_4 and p_11 happen to target the same downstream hub,
their response vectors (dominated by firing from that hub) will be
correlated. The small cluster {4, 5, 6, 11} is the observed footprint
of such shared downstream structure.

## What this means

- **Snap on real W works for most input pairs.** The 12/16 "easy"
  classes confirm that when two inputs drive disjoint downstream sets,
  argmax cosine recovers the correct prototype at the stability
  ceiling, as expected.
- **Snap fails when inputs share downstream structure.** With random
  input patterns, this happens ~1/16 of the time on this sample. The
  failure is predictable and diagnosable from the gap: any class with
  a gap below ~0.1 is at risk, and the 4/16 classes with small gaps
  are exactly the cluster where the single flip occurred.
- **The fix is biologically-structured inputs, not more noise control.**
  Using labeled PN / KC / MBON populations would give input patterns
  whose downstream targets are *designed* to be non-overlapping (that
  is what the PN→KC expansion does in the real fly). Random 50-neuron
  patterns are the worst-case for snap on real W; biological patterns
  will be much easier.

## Scope of the claim, honestly stated

- 16 classes, single sample per class, n=1 seed per query. A proper
  evaluation would be multi-seed per class.
- Random inputs, not biologically-structured ORN/PN patterns. The
  failure mode this exhibited (random-pattern overlap) is not the
  failure mode that matters for the paper's MB-based conditional.
- 100 ms window. Longer windows let recurrence distribute activity
  more widely, which may either fix the overlap (high-dim response)
  or make it worse (attractor collapse) — needs sweep.

## Implications for the paper

The polar-decomp fuzzy-conditional result (n=35 × 16 decisions = 560/560)
is stronger than this real-W 15/16, but the polar-decomp result is
under the category-error of running arithmetic that does not actually
happen on the substrate. The real-W 15/16 is weaker on paper but
stronger in claim: it is what the substrate actually does.

The paper-ready strategy is:
1. Keep the 560/560 polar-decomp result as the clean-math reference.
2. Report the 15/16 real-W snap as the substrate-truth result.
3. Say honestly that the gap between the two is what happens when
   you ground the operation in real biology: the math gets noisier
   but the claim becomes substrate-executable.
4. Flag that biologically-structured inputs (not random) are the path
   to closing the gap.
