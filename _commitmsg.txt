rust variant inner match + nested if let: hoist int-locals from match-arm bodies

The Rust frontend already hoisted a top-level expression-position match, but a VARIANT
match or if-let NESTED inside another match's ARM hit UNSUPPORTED -- the tail-match path
called _lower_match_stmts without running the hoist walk on arm bodies. Fix: (1)
_lower_match_stmts now runs _hoist_enum_constructions on each arm result and emits the
hoisted int-local prelude after the outer _vtag/_val decls; (2) added an if-let branch
to _hoist_enum_constructions that hoists int _vtag_il{k} = realvec(scrut.item("_tag"))
and registers the crisp-tag blend at the use site (the function-tail if-let recipe).
Both reuse the existing _ARG_HOIST use-site resolution.

Fixtures nested_variant_match_arm (f(A(3),B(5)) = 3+(0-5)) RUN == -2.0 and
nested_if_let_arm (f(A(3),A(5)) = 3+5) RUN == 8.0, both == ground truth on the substrate.
Rust suite 54 passed, no regressions. Rust now has no open WASM-fallback items; the shared
int-local limit is fully cleared across Haskell + Rust.
