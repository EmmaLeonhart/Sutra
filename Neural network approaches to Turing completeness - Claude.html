Your three categories are solid, but the landscape is richer. Here's how I'd extend it:

Within your framework (things you may be underweighting)

Your category 3 (external memory) actually splits meaningfully:

  • Differentiable Neural Computers / NTMs — addressable tape-like memory
  • Stack-augmented networks — push/pop stacks, which are Turing complete with just one stack (vs. needing two for a full Turing machine, though one suffices with the right formalism)
  • Pointer networks / attention over mutable state — a weaker version but sometimes analyzed in this vein

Additional distinct proposals

  1. Hypernetworks as dynamic weight generation — a network that writes the weights of another network at runtime. If the weight-writing process is unconstrained enough, the inner network's program is being "written" step by step, which is a different Turing completeness route than recurrence or memory.
  2. Neural program synthesis / execution — networks trained to explicitly simulate a register machine or lambda calculus (e.g. Neural Turing Machines are here, but also work like Graves' differentiable Forth). The Turing completeness comes from the target formalism being compiled into, not the network architecture per se.
  3. Unbounded depth via recursive/iterative application — if you allow a fixed network to be applied an unbounded number of times with its own output as input (and some halting condition), you get Turing completeness without recurrent weights per se. This is the sense in which "thinking tokens" / chain-of-thought with a scratchpad can be analyzed — the network is finite but the process is not.
  4. Continuous-time dynamics / ODEs — Neural ODEs and related continuous dynamical systems. Siegelmann & Sontag's original result was actually about analog recurrent nets (with real-valued weights), which are strictly more powerful than Turing machines. Finite-precision discrete RNNs hit exactly Turing completeness; real-valued ones exceed it. This is a distinct axis from architecture.

The axis your taxonomy is missing

Your three categories are distinguished by where the unbounded resource lives:

  • Recurrence → unbounded time (steps)
  • Reservoir → unbounded state (fixed dynamics, but infinite reservoir = infinite state)
  • External memory → unbounded space (tape/stack/context)

The cleaner organizing principle might be: what resource is being made unbounded? Time, state, and space give you the three classical routes, but you can also make precision unbounded (analog nets) or depth unbounded (iterative application), which are genuinely different.

The Siegelmann & Sontag distinction — which you've used for Sutra's theoretical framing — is really making the precision axis explicit, which is what makes it sharper than just "it's an RNN."
